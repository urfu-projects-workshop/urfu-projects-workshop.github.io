Сборка проекта

Инструкции по сборке проекта

Содержание

  • Сборка через GitHub Actions
  • Локальная сборка
    • Серверная часть
    • Клиентская часть

Собрать проект можно двумя способами:

  • Использовать готовый сценарий сборки в GitHub Actions
  • Собрать проект локально

Далее будет подробно описан каждый из вариантов.

Сборка через GitHub Actions #

Для начала необходимо зайти в репозиторий с серверной частью проекта (workshop-backend) на платформе GitHub.
На странице с репозиторием необходимо выбрать вкладку Actions. Сейчас в проекте есть один настроенный сценарий, который выполнит сборку клиентской и серверной части системы. Результат сборки можно будет скачать в виде архива. Для этого необходимо выполнить следующие шаги:

  1. На вкладке Actions нужно выбрать сценарий (workflow) Build.
  2. Нужно указать параметры для сборки проекта:
    • ветку для сборки серверной части
    • ветку для сборки клиентской части *Для сборки клиентской части используется другой репозиторий. Для доступа к нему GitHub требует специальный ключ. Для текущего репозитория он уже настроен. Если проект будет перенесен, то нужен будет новый ключ, который должен храниться в GitHub Secrets для репозитория.
    • режим сборки (production/development)
    • необходимость генерации скрипта миграций для БД.
  3. Необходимо запустить сборку проекта. Для этого нужно нажать кнопку “Run workflow”.

Три первые шага представлены на скриншоте ниже:

  1. Нужно дождаться успешного завершения сборки проекта. Пример успешной сборки представлен ниже на скриншоте:
  2. Требуется скачать собранные файлы проекта в виде архиве. Их можно найти в нижней части страницы в разделе с артефактами сборки.
    Для удобства при сборке генерируется три артефакта:
    • клиентская часть системы (fronted-build-output)
    • серверная часть системы (backend-build-output)
    • общий архив (build-output)

Раздел Artifacts представлен на скриншоте ниже:

На этом сборка проекта завершена. Полученный архив нужно будет загрузить на виртуальную машину, на которой запущена система. Подробнее об этом будет сказано далее в разделе “Развертывание в продуктовом окружении” (находится в инструкции в Google Docs).

Локальная сборка #

Локальная сборка проекта состоит из двух частей: сборка клиентской части и сборка серверной части. Далее будут рассмотрены обе части.

Серверная часть #

Для сборки серверной части проекта у вас должна быть настроена локальная среда разработки. Подробная информация, как это сделать есть в инструкции по запуску backend проекта.
Разберем алгоритм сборки серверной части проекта по шагам:

  1. Откроем проект в среде интегрированной разработки Visual Studio 2022 (можно использовать и более ранние версии, алгоритм будет аналогичным).
  2. выберем проект Workshop.Web и кликнем по нему правой кнопкой мыши. В появившемся выпадающем списке на нужно кликнуть на Publish.
    Пример есть на скриншоте ниже:
  3. Необходимо настроить параметры сборки проекта.
    Рекомендуется выставить настройки как на скриншоте ниже:
  4. Необходимо нажать кнопку Publish и дождаться успешной сборки проекта. Пример успешной сборки представлен на скриншоте ниже:
  5. (Optional) Создать скрипт миграции для базы данных.
    Для это можно выполнить одну из команд ниже.

Создание скрипта миграции через Visual Studio Package Manager Console:

Script-Migration -Idempotent -Output .\BuildOutput\migration.sql

Создание скрипта миграции через командную строку:

dotnet ef migrations script --idempotent --output .\BuildOutput\migration.sql

На этом сборка серверной части проекта завершена. Далее рассмотрим сборку клиентской части проекта.

Клиентская часть #

Для сборки клиентской части проекта у вас должна быть настроена локальная среда разработки. Подробная информация, как это сделать есть в инструкции по запуску frontend проекта.
Разберем алгоритм сборки клиентской части проекта по шагам:

  1. Откроем проект в среде интегрированной разработки Visual Studio Code (можно использовать другую IDE, алгоритм будет аналогичным).
  2. Откроем терминал и запустим сборку командой:
    • npm run build - development сборка
    • npm run build:prod - production сборка
  3. Необходимо дождаться успешного завершения сборки.
    Все артефакты сборки будут лежать в папке “build”.

На этом сборка клиентской части системы завершена. После этого нам необходимо объединить собранные артефакты серверной и клиентской части в один архив.
Для этого мы должны скопировать содержимое папки “build” (артефакты клиентской части) и перенести их в папку “wwwroot” в директории со сборкой серверной части.
Теперь мы готовы к загрузке системы на виртуальную машину.

Данная информация находится разделе “Развертывание в продуктовом окружении” из инструкции в Google Docs.
